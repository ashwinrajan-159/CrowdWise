Reading this board. Score = predicted delay (min) × population exposure; the bar shows each chokepoint's severity. The system suggests officers by severity — the monitor assigns the final number per chokepoint with the +/− stepper (no fixed pool). Each prediction lists the most similar past events it was built from; if those analogs look wrong, adjust.

What "refresh" does. It re-scrapes the upcoming event calendar and re-forecasts — it updates event forecasts, not live traffic state. A real connected-vehicle feed is the designed-for next phase.

Planned vs. unplanned. The live feed covers planned events — concerts, sports, processions, roadworks — which is the wedge: the city knows about them days ahead. Unplanned incidents (accidents, breakdowns) can't be forecast from a calendar; they need a live incident feed and are a Phase-2 source. The model has still learned from historical unplanned events, so an uploaded incident is scored too.
